Week 13: Multiple Choice Questions

Test your knowledge on Grouping Data (GROUP BY, HAVING) and Modifying Data (INSERT, UPDATE, DELETE, TRANSACTIONS).

1. PostgreSQL evaluates the `HAVING` clause AFTER which of the following clauses?
  • SELECT and DISTINCT
  • ORDER BY and LIMIT
  • FROM, WHERE, and GROUP BY
  • Only the FROM clause
Answer: c) FROM, WHERE, and GROUP BY
PostgreSQL evaluates the HAVING clause after the FROM, WHERE, and GROUP BY clauses, and before the DISTINCT, SELECT, ORDER BY, and LIMIT clauses.
2. Why can't you use column aliases defined in the `SELECT` clause within the `HAVING` clause?
  • Because column aliases are not supported in PostgreSQL.
  • Because the HAVING clause is evaluated before the SELECT clause.
  • Because the HAVING clause only works with the COUNT() function.
  • Because column aliases can only be used in the WHERE clause.
Answer: b) Because the HAVING clause is evaluated before the SELECT clause.
At the point of HAVING clause evaluation, the column aliases specified in the SELECT clause are not yet available.
3. What happens if you omit the `WHERE` clause in an `UPDATE` statement?
  • The statement throws a syntax error.
  • Only the first row in the table is updated.
  • All rows in the table will be updated.
  • No rows are updated.
Answer: c) All rows in the table will be updated.
The WHERE clause targets specific rows. If it is omitted, the condition applies to all rows in the table.
4. Which PostgreSQL-specific feature allows you to return the data generated by an insert (such as an auto-incremented ID) without requiring a separate SELECT query?
  • OUTPUT clause
  • YIELD clause
  • RETURNING clause
  • FETCH NEXT clause
Answer: c) RETURNING clause
The RETURNING clause immediately returns the inserted, updated, or deleted rows.
5. What does the hidden `ctid` column in PostgreSQL represent?
  • The creation timestamp of the row.
  • The physical location of the row version.
  • The transaction ID that last modified the row.
  • The primary key of the table.
Answer: b) The physical location of the row version.
It is the simplest way to distinguish between two rows that have identical data, making it highly useful for deleting duplicate rows.
6. Which command is used to undo all changes made since the `BEGIN` command in a transaction?
  • COMMIT
  • SAVEPOINT
  • REVERT
  • ROLLBACK
Answer: d) ROLLBACK
ROLLBACK cancels the entire process to prevent partial data entry when an error occurs or a mistake is recognized.
