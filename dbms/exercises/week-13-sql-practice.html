Week 13: SQL Practice Exercises

Local Table Setup Script

Run the following script in SQL Studio or pgAdmin to create the environment required for the exercises.

-- Cleanup existing schema DROP TABLE IF EXISTS order_details; DROP TABLE IF EXISTS orders; DROP TABLE IF EXISTS products; -- 1. Create Tables CREATE TABLE products ( product_id SERIAL PRIMARY KEY, product_name VARCHAR(100) NOT NULL, category VARCHAR(50), unit_price NUMERIC(10, 2) ); CREATE TABLE orders ( order_id SERIAL PRIMARY KEY, customer_id INT NOT NULL, order_date DATE DEFAULT CURRENT_DATE, status VARCHAR(20) ); CREATE TABLE order_details ( detail_id SERIAL PRIMARY KEY, order_id INT REFERENCES orders(order_id) ON DELETE CASCADE, product_id INT REFERENCES products(product_id), quantity INT CHECK (quantity > 0) ); -- 2. Insert Seed Data INSERT INTO products (product_name, category, unit_price) VALUES ('Steel Sheet', 'Raw Material', 100.00), ('Copper Wire', 'Raw Material', 50.00), ('Gear Assembly', 'Component', 250.00), ('Motor', 'Component', 500.00), ('Welding Rod', 'Consumable', 15.00); INSERT INTO orders (customer_id, order_date, status) VALUES (501, '2024-01-10', 'Shipped'), (502, '2024-01-12', 'Pending'), (501, '2024-01-15', 'Shipped'), (503, '2024-02-01', 'Shipped'); INSERT INTO order_details (order_id, product_id, quantity) VALUES (1, 1, 10), (1, 3, 2), (2, 4, 1), (3, 2, 20), (3, 3, 5), (4, 5, 100);
Q1. Basic Update

Write a query to change the status of order 101 (or 1 in our local table) to 'Delivered'.

UPDATE orders SET status = 'Delivered' WHERE order_id = 1;
Q2. Conditional Mass Update

Adjust the unit_price of all 'Raw Material' products by subtracting 5.00 from their current price.

UPDATE products SET unit_price = unit_price - 5.00 WHERE category = 'Raw Material';
Q3. Multi-column Update with Verification

Change the product_name to 'Heavy Motor' and unit_price to 600.00 for the product with product_id = 4. Return the updated row data.

UPDATE products SET product_name = 'Heavy Motor', unit_price = 600.00 WHERE product_id = 4 RETURNING *;
Q4. Basic Delete

Write a query to delete the product with product_id = 3.

DELETE FROM products WHERE product_id = 3;
Q5. Filtered Delete

Remove all orders that have a status of 'Cancelled'.

DELETE FROM orders WHERE status = 'Cancelled';
Q6. Delete with Using

Delete all rows from the order_details table that are associated with customer_id = 501. (Note: customer_id is in the orders table).

DELETE FROM order_details USING orders WHERE order_details.order_id = orders.order_id AND orders.customer_id = 501;
Q7. Using GROUP BY and HAVING

Write a query using the GROUP BY and HAVING clauses to find the total sum of the unit_price of products grouped by their category. Only return groups where the total sum is greater than 100.

SELECT category, SUM(unit_price) FROM products GROUP BY category HAVING SUM(unit_price) > 100;
Q8. Transaction Rollback

Write a transaction block that attempts to update a product price to an incorrect amount, and then completely cancels the transaction using a Rollback.

BEGIN; UPDATE products SET unit_price = unit_price * 10 WHERE category = 'Component'; ROLLBACK;
